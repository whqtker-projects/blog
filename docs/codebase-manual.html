코드베이스 매뉴얼

이 문서는 현재 Astro 기반 블로그 저장소를 이해하고 작업하기 위한 한국어 매뉴얼이다. 사이트 기능을 추가하는 문서가 아니라, 저장소 구조와 작업 흐름을 빠르게 파악하기 위한 운영 참고 문서다.

관련 문서 맵은 docs/README.md에서 확인한다.

저장소가 하는 일

이 저장소는 Obsidian vault에서 작성한 기술 글을 Astro 정적 사이트로 빌드한다. Obsidian은 원본 작성 공간이고, 이 저장소의 src/content/posts/는 변환되어 커밋되는 산출물이다. 

Obsidian vault
  -> pnpm convert --input <vault/posts> --strict
  -> src/content/posts/
  -> pnpm build
  -> dist/

pnpm build는 Obsidian vault를 읽지 않는다. 빌드는 저장소에 이미 커밋된 Markdown과 source code만 사용한다. vault 변경사항을 반영하려면 먼저 pnpm convert를 명시적으로 실행해야 한다.

주요 디렉터리

경로 역할
src/pages/ Astro route를 정의한다. 홈, series, post, example 페이지가 여기에 있다.
src/layouts/ 공통 HTML shell과 post/example page wrapper를 둔다.
src/components/ 반복되는 UI 조각을 둔다. 현재는 series list와 example list가 있다.
src/content/ Astro content collections의 실제 Markdown과 attachment를 둔다.
src/content/examples/ post에 부속되는 example 문서를 수동 작성하는 위치다. 각 example은 post frontmatter로 정확히 하나의 post에 연결된다.
src/content/series_indexes/ parent/child series index를 둔다. 경로 구조와 frontmatter가 함께 series 계층을 정의한다.
src/utils/ 정렬, visibility, series hierarchy, remark 변환 helper를 둔다.
scripts/ Obsidian 변환, content validation, series graph metadata sync 같은 repository 작업 도구를 둔다.
docs/ 운영 문서, 결정 기록, workflow 문서를 둔다. 현재 기준 문서는 archive 밖에 있는 문서다.
public/images/ content collection 바깥에서 직접 참조할 정적 이미지 자산을 둔다.
docs/archive/ 과거 계획과 superseded 문서를 보관한다. 새 작업의 기준 문서로 사용하지 않는다.

Route 구조

URL 파일 역할
/ src/pages/index.astro parent series 목록을 보여주는 홈이다.
/series/[parent] src/pages/series/[parent].astro parent series 설명과 child series 목록을 보여준다.
/series/[parent]/[child] src/pages/series/[parent]/[child].astro child series에 속한 visible post 목록을 보여준다.
/posts/[slug] src/pages/posts/[slug].astro 개별 post를 렌더링한다. prev/next navigation은 같은 child series의 order를 기준으로 한다.
/posts/[slug]/examples/[example] src/pages/posts/[slug]/examples/[example].astro post에 붙은 project-style example을 별도 페이지로 렌더링한다.

콘텐츠 모델

posts

개념을 깊게 설명하는 본문이다. 파일은 src/content/posts/에 있으며, Obsidian vault의 posts/에서 변환된다. 각 post는 정확히 하나의 child series에 속한다.

examples

긴 구현 예시를 post 본문 밖으로 분리한 수동 작성 문서다. 파일은 src/content/examples/ 바로 아래에 두며, frontmatter의 post로 owning post를 가리킨다.

series_indexes

parent series와 child series를 정의한다. parent index는 root에 두고, child index는 src/content/series_indexes/<parent>/<child>.md에 둔다.

자세한 정보 구조 계약은 docs/content-model.md를 기준으로 한다.

Frontmatter 계약

실제 schema는 src/content.config.ts가 정의한다. 문서나 예시가 schema와 다르면 src/content.config.ts와 validation script를 기준으로 판단한다.

Collection 필수 필드 선택 필드
posts title, series, order, status description, tags
examples title, post, order, status description
series_indexes title, series parent, order, description, aliases, tags

status는 idea, draft, published만 허용한다. local dev는 unpublished content를 볼 수 있지만 staged/production build는 published만 노출한다.

추가로 repository validation은 post의 series가 child series를 가리키는지, example의 post가 실제 post slug를 가리키는지, child series의 parent와 order가 맞는지, 숫자 제목 prefix가 있으면 명시적 order와 일치하는지까지 검사한다.

변환과 검증

자주 쓰는 명령

명령 사용 시점
pnpm install 처음 checkout했거나 dependency가 바뀌었을 때 실행한다.
pnpm dev local Astro dev server를 띄운다.
pnpm preview pnpm build 결과물을 production과 가까운 형태로 로컬에서 확인한다.
pnpm convert --input <vault/posts> --strict Obsidian post 변경을 repository content로 동기화한다.
pnpm check:content series hierarchy, status, order, example attachment 같은 repository content 계약을 검사한다.
pnpm test:convert Obsidian-to-Astro converter를 수정했을 때 실행한다.
pnpm test:repo content helper, ordering, visibility, series validation, remark plugin을 수정했을 때 실행한다.
pnpm build schema, route, layout, content 변경 후 production build를 확인한다.

변환 세부 계약은 docs/obsidian-conversion-contract.md, publishing 흐름은 docs/publishing-workflow.md를 따른다.

흔한 작업별 위치

작업 수정 위치 검증
새 post 추가 Obsidian vault에서 작성 후 pnpm convert로 src/content/posts/에 반영 pnpm check:content, pnpm build
새 example 추가 src/content/examples/에 수동 작성하고 post frontmatter로 owning post 지정 pnpm check:content, pnpm build
새 series 추가 src/content/series_indexes/에 parent/child index 작성 pnpm check:content, pnpm build
변환기 수정 scripts/obsidian-to-astro.mjs와 관련 test pnpm test:convert, pnpm build
content validation 수정 scripts/check-content.mjs와 helper/test pnpm test:repo, pnpm check:content
route 또는 UI 수정 src/pages/, src/layouts/, src/components/, src/styles/ pnpm build
문서 수정 docs/와 필요한 경우 CLAUDE.md, AGENTS.md 상대 링크 검사, 관련 repository 검증

주의 규칙

  • status: published인 post는 사용자 명시 요청 없이 수정하지 않는다.
  • src/content/posts/는 변환 산출물이지만 git에 커밋되는 source of truth다. 직접 수정이 필요하면 원본 또는 converter와의 관계를 먼저 확인한다.
  • post filename과 slug는 lowercase English kebab-case를 유지한다. 한국어 제목은 title에 둔다.
  • docs/archive/는 보관소다. 현재 작업 기준은 archive 밖 활성 문서에서 찾는다.
  • series의 구조적 순서는 frontmatter의 order가 기준이다. 제목 숫자 prefix는 표시 보조일 뿐이다.
  • new structural decision이 필요하면 추정으로 확정하지 말고 docs/open-questions.md와 decision workflow를 따른다.
